# Notion Block Patterns

Use Notion blocks to lower reading cost. Do not add structure that has no job.

## Block Selection
| Need | Preferred block |
|---|---|
| Page purpose or executive takeaway | Short paragraph or callout |
| Navigation | H2/H3 headings |
| Comparison | Table |
| Decisions | Decision table or short callout plus rationale |
| Action tracking | Checklist or task table |
| Optional detail | Toggle |
| Risks | Table with impact, likelihood, mitigation, owner |
| References | Bulleted links or source table |
| Related work | Linked pages section |

## Page Shape
- Put the summary, decision, or recommendation above supporting detail.
- Keep first-screen content short enough for a busy reader.
- Use tables only when rows share the same fields.
- Use toggles for appendices, raw notes, and detailed source excerpts.
- Avoid deeply nested headings in short pages.
- Name links descriptively; avoid "click here".
