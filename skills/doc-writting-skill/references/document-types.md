# Document Types

Choose the document type before drafting. If the requested page combines types, name the primary type and include secondary sections only when useful.

## Product Spec
Use for features, systems, experiments, or product changes.

Recommended sections:
- Summary
- Problem
- Goals and non-goals
- Users and scenarios
- Requirements
- Proposed solution
- Alternatives considered
- Risks and mitigations
- Open questions
- Rollout or next steps

Use tables for requirements, decisions, and rollout phases.

## Decision Record
Use when a choice needs to be preserved and discoverable.

Recommended sections:
- Decision
- Status
- Context
- Options considered
- Rationale
- Consequences
- Follow-ups
- Related pages

The decision must be one sentence near the top.

## Research Brief
Use for synthesizing sources into findings.

Recommended sections:
- Question
- Short answer
- Key findings
- Evidence table
- Contradictions or uncertainty
- Implications
- Sources
- Open questions

Separate evidence from interpretation.

## Proposal
Use for recommending a path that needs approval.

Recommended sections:
- Recommendation
- Why now
- Expected impact
- Plan
- Costs and tradeoffs
- Risks
- Decision needed
- Appendix

State the decision requested from the reader.

## SOP or How-To
Use for repeatable operational work.

Recommended sections:
- Purpose
- Scope
- Prerequisites
- Steps
- Edge cases
- Troubleshooting
- Owner
- Last reviewed

Use numbered steps for sequences and checklists for verification.

## Meeting-Derived Document
Use when turning notes/transcripts into durable knowledge.

Recommended sections:
- Summary
- Decisions
- Action items
- Discussion notes
- Risks or blockers
- Open questions
- Source meeting

Do not preserve chronology unless sequence matters.

## Knowledge Base Page
Use for durable explanations, concepts, and onboarding.

Recommended sections:
- What this is
- When to use it
- Key concepts
- Examples
- Related pages
- Maintainer
- Last reviewed

Optimize for future retrieval, not just current readability.
