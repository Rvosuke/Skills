# Writing Standards

Use these standards when writing or revising Notion documents.

## Reader Contract
Every page should make four things clear:

| Question | What the doc must provide |
|---|---|
| Where am I? | A precise title, short opening summary, and scannable headings |
| Why am I here? | Purpose, audience, and decision/use context |
| What matters? | Main claims, decisions, requirements, or takeaways |
| What happens next? | Actions, owners, open questions, links, or archive status |

## Style Calibration
If the user provides past writing samples or an existing page to match:
1. Extract sentence rhythm, paragraph length, preferred terms, formality, and directness.
2. Apply the profile as a soft guide.
3. Let document purpose and audience override personal style.
4. If the sample is weak or too sparse, say the profile is partial and apply cautiously.

## Prose Rules
- Start with the actual point. Avoid "this document will discuss" unless the page is explicitly a table of contents.
- Prefer concrete nouns and verbs. Replace inflated terms with precise terms.
- Keep terminology stable. Do not cycle synonyms for the same concept within one page.
- Vary paragraph length by function. Short paragraphs emphasize decisions; longer paragraphs carry reasoning.
- Use hedging for inference, forecasts, and incomplete evidence.
- Avoid unsupported certainty, generic "best practice" claims, and decorative language.

## Common Anti-Patterns
| Pattern | Fix |
|---|---|
| Vague importance words such as "crucial", "robust", "comprehensive" | State the concrete effect or criterion |
| Throat-clearing openers | Delete and begin with the claim |
| Uniform three-part structures | Use the number of points the evidence supports |
| Long prose where a table is better | Convert comparisons, owners, risks, timelines, and requirements to tables |
| Action items hidden in paragraphs | Convert to checklist or task table |
| Confident prose over missing evidence | Mark the gap and ask for source material |

## Section-Level Test
For each section, ask:
- Does this section have one clear job?
- Would deleting it break the reader's understanding?
- Does it contain evidence, decision, or action content?
- Is the next step clear?

Delete, merge, or shorten sections that fail the test.
