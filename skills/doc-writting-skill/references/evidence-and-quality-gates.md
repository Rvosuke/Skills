# Evidence and Quality Gates

Use this reference whenever the document makes factual claims, recommendations, or decisions.

## Evidence Status
| Status | Meaning | How to write it |
|---|---|---|
| Source-backed | Verified in a fetched page, linked source, or provided material | Cite/link the source |
| User-declared | Stated by the user but not independently verified | Mark as user-provided if material |
| Inference | Reasoned from available evidence | Say "This suggests", "likely", or "inference" |
| Missing | Needed but unavailable | Mark `TODO: verify` or `Evidence gap` |
| Conflicting | Sources disagree | Name both sources and describe the conflict |

## Evidence Ledger
Create a lightweight ledger for non-trivial pages:

| Claim or decision | Evidence status | Source | Gap or caveat |
|---|---|---|---|

The ledger can be internal during drafting or visible as an appendix for research, proposals, and decisions.

## Anti-Leakage Rule
When the user provides source material, base factual content on that material. Use general writing knowledge for structure and clarity, but do not invent:
- metrics
- dates
- owners
- decisions
- policies
- meeting outcomes
- technical constraints
- references

If the source material does not cover a needed point, mark the gap instead of filling it from memory.

## Publish Gate
Before creating or updating a Notion page, verify:
- The first section states the purpose or takeaway.
- The intended audience is clear from title, intro, or context.
- Every important claim has an evidence status.
- Recommendations explain tradeoffs.
- Decisions state rationale and consequences.
- Action items have owner/status/date when known.
- Open questions are not hidden in prose.
- Related pages or sources are linked.
- The page title is specific and searchable.

If the gate fails, fix the page or add a visible `Open Questions` / `Evidence Gaps` section.
