# Revision Protocol

Use this when improving an existing Doc page.

## Intake
Fetch the current page and identify:
- Purpose of the revision
- Intended audience
- Sections to preserve
- Sections that are outdated, unclear, duplicated, or unsupported
- Whether the task is copyediting, restructuring, fact correction, or substantial rewrite

## Revision Levels
| Level | Use when | Output |
|---|---|---|
| Light edit | Prose is mostly right | Cleaner wording and small structure fixes |
| Structural edit | Reader path is unclear | New outline and rewritten section order |
| Evidence edit | Claims need support | Evidence ledger, gap labels, source links |
| Rewrite | Page no longer serves its purpose | New draft with a change summary |

## Rules
- Preserve decisions unless the user asks to revisit them.
- Preserve source links and page relations.
- Do not delete appendices, child pages, or databases without explicit user approval.
- Keep unresolved questions visible.
- When editing in Doc, use exact old/new replacement strings from the fetched page.

## Change Summary
For substantial edits, add or report:

| Area | Change | Reason |
|---|---|---|

Use the summary to help future readers understand why the page changed.
